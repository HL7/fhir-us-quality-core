US Quality Core addresses provenance at the level of data elements defined in each profiled FHIR resource, for example, author, performer, authored or issued date, occurrence date, and other elements that indicate who did what, when, and under what context. This follows the current [US Core “Basic Provenance”](https://hl7.org/fhir/us/core/basic-provenance.html) guidance for addressing individual-level provenance, in which activities performed by specific actors such as patients or providers and are captured using resource-specific data elements rather than separate FHIR Provenance resources. This contrasts with using FHIR [Provenance](http://hl7.org/fhir/us/core/STU9/StructureDefinition-us-core-provenance.html) resource to track metadata about events such as creation and modification of a resource instance. If US Core updates its approach to provenance, US Quality Core will align with those updates.

US Core maps the US Core data elements that communicate the author and author roles corresponding to the [U.S. Core Data for Interoperability (USCDI)](https://isp.healthit.gov/united-states-core-data-interoperability-uscdi) Provenance Author and Author Role data elements requirements. The US Quality Core mappings below extend this same individual-level provenance pattern to the US Quality Core profiles that lack corresponding US Core profiles and therefore are not covered by the US Core guidance table. For each row, the Data Element Corresponding to Author Provenance identifies the resource element that conveys the author and maps to the USCDI+ Quality Author, and the Target Resource Types Corresponding to Author Role Provenance identifies the referenced resources that maps to the USCDI+ Author Role.
